Getting Started

Introduction

The GRL Qi Wireless Charger Tester (GRL-WP-BST-C3, the following is called C3) is a powerful Qi compliance tester that has been approved by the WPC. You can use GRL's C3 software for full compliance testing. For advanced users, can be directly called through the API to meet more customized test requirements

Prerequisites

Following are the prerequisites for using the software:

  • This document assumes that users of API have knowledge on WPC Wireless Power Specification
  • Knowledge of C# or Python to use the API
  • Visual Studio or any other IDE's
  • Laptop or PC with Windows 10 with minimum of 8GB of RAM
  • GRL-WP-BST-C3 Hardware
  • GRL-WP-BST-C3 software installed on the PC / Laptop
  • UUT (Unit under test)

Architecture

Custom test cases were written in C# as independent standalone application. These test cases call the API's defined in GRLC3ApiLib.dll and all the support functions and helper classes are defined in GrlQiLib.dll.

Hardware Setup

Following diagram illustrates the required hardware connection for using the API's along with the GRL-WP-BST-C3:


The GRL-WP-BST-C3 Software is loaded on a Windows 10 computer. Below is a procedure for connecting the hardware and computer.

  1. Connect the GRL-WP-BST-C3 Power Interface using the 24V, 280W Power Brick included with the controller.
  2. Connect the GRL-WP-BST-C3 Controller using a physical Ethernet connection between the computer and the controller.
  3. The Ethernet port on the control computer needs to be configured correctly for the GRL-WP-BST-C3 controller to recognize the control computer and vice versa.
  4. To make sure the network connection is set up correctly, goto Network Connections from the Control Panel on the computer.
  5. Open the Ethernet properties settings for the Ethernet port that will connect to the GRL-WP-BST-C3 controller, select “Internet Protocol Version 4 (TCP/IPv4)” and click on the “Properties” button below and to the right.
  6. Set up the TCP/IPv4 properties as shown below and select “Use the following IP address:” to configure static IP address as 192.168.255.n where n is any number between 2 and 255. The subnet mask should be set as 255.255.255.0 and the default gateway as 192.168.255.1. The rest of the items will remain unchanged.
  7. Once configured click the “OK” button and close the TCP/IPv4 Properties pop-up. Make sure the GRL-WP-BST-C3 Controller is powered on and completely booted up (front panel display showing firmware version number) and then connect the Ethernet cable from the GRL-WP-BST-C3 controller to the computer’s Ethernet port that was just set up. The Network Connections window should now appear as in the below image

  8. The GRL-WP-BST-C3 Controller is now set up and ready for use.

  9. Before running any tests, it is recommended that you verify both the control computer and the GRL-WP-BST-C3 Controller are communicating properly by using ping 192.168.255.1 from command line. <\ol>

Software Setup

You can create custom test cases using C#. To create API programming you need the following software:

  • Python 2.7.15 (Windows x86 MSI installer) or Visual Studio 2017
  • GRL-WP-BST-C3 Software


  1. If you want to use C# to create the custom test cases install Visual Studio 2019 from https://visualstudio.microsoft.com/downloads/
    Download the appropriate Visual Studio installer version and follow the on screen instructions to install the software.
  2. All API libraries and helper functions are installed along with the GRL-WP-BST-C3 software. Download the GRL-WP-BST-C3 software from https://graniteriverlabs.com/download-center/ and follow the on screen instructions to install the software.

Development Environment Setup

Creating custom test cases in C#

To develop the application in C#, follow the below instructions:

  1. Open Visual Studio 2017 or higher.
  2. Create a console application in Visual Studio wizard from the [File] menu, select [New]-[Project…], and click on Windows Forms App (.NET Framework) or "Console App (.NET Framework)".
    <BR>
  3. Add references to GRLC3ApiLib.dll and GrlQiLib.dll under the project "Reference".

  4. You are now set for C# application development; refer to the sample code documentation for further details.

Creating custom test cases in Python

To develop the application in Python, follow the below instructions:

  1. Download and install version 2.7.15 of Python (Windows x86 MSI installer) from https://www.python.org/downloads/release/python-2715/

  2. Launch GRL-WP-BST-C3 Software from desktop icon "GRL-WP-BST-C3 Software"
  3. Navigate to Connection configuration panel and click on "Update Firmware" button and follow on screen instructions to upgrade the firmware
  4. Close the GRL-WP-BST-C3 Software
  5. Launch Python utility from desktop icon "GRL-WP-BST-C3 Python API Tool" or from API installation folder path C:\GRL\GRL-WP-BST-C3\APILibrary Python API Tool
  6. Refer existing Python sample scripts in C:\GRL\GRL-WP-BST-C3\APILibrary\C# Scripts - Python

  7. If you find any issue in running Python scripts, you may try below steps and if still issue exist please report it to suppo.nosp@m.rt@g.nosp@m.ranit.nosp@m.eriv.nosp@m.erlab.nosp@m.s.co.nosp@m.m
  8. Add following Python installation paths to the system PATH variable C:\Python27; C:\Python27\Scripts ; C:\Python27\Lib.
  9. Download get-pip.py from https://pip.pypa.io/en/stable/installing/ and copy to C:\Python27.
  10. Open command prompt and navigate to C:\Python27. Type python get-pip.py to install PIP.

  11. Install pythonnet via PIP by using the "pip install pythonnet" command in command prompt.

  12. You are now set for Python application development; refer to the sample code documentation for further details.
